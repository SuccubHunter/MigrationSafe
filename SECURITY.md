# Security Policy

## Supported Versions

We provide security updates for the following versions:

| Version | Supported               |
| ------- | ----------------------- |
| 0.4.x   | :white_check_mark: Yes  |
| 0.3.x   | :white_check_mark: Yes  |
| < 0.3   | :x: No                  |

## Reporting a Vulnerability

If you discover a security vulnerability, please **DO NOT** create a public issue.

Instead, send an email to: **succubhuntersinc@gmail.com**

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggestions for a fix (if available)

### What We Will Do

1. Acknowledge receipt within 48 hours
2. Assess the vulnerability and its severity
3. Develop a fix
4. Release a patch for all supported versions
5. Publish a security advisory after the fix is released

### Timelines

- **Critical vulnerability**: fix within 7 days
- **High severity vulnerability**: fix within 30 days
- **Medium/Low severity vulnerability**: fix in the next release

## Acknowledgements

We thank everyone who responsibly reports security vulnerabilities. The names of security researchers will be included in the security advisory (with their consent).

## Known Vulnerabilities

A list of known vulnerabilities and their status is published in [GitHub Security Advisories](https://github.com/SuccubHunter/migsafe/security/advisories).

**Note:** If you do not have access to email for security reporting, create a private security advisory via the GitHub Security tab in the repository.
